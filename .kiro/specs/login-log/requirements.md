# 需求文档

## 简介

本文档规定了 WYH Admin 项目系统模块中登录日志功能的需求。登录日志系统跟踪所有用户登录尝试（成功和失败），包含用户身份、网络位置、设备详情和登录状态等全面信息。系统支持多租户隔离、可配置的数据留存和灵活的查询能力。

## 术语表

- **登录日志系统（Login_Log_System）**：负责记录、存储和管理用户登录尝试记录的子系统
- **登录请求（Login_Request）**：用户提交凭证发起的身份验证尝试
- **登录状态（Login_Status）**：登录尝试的结果（成功或失败）
- **租户（Tenant）**：多租户系统中的隔离组织单元
- **超级管理员（Super_Admin）**：具有跨租户访问权限的特权用户
- **留存期限（Retention_Period）**：登录日志记录在自动删除前保留的时长
- **地理位置（Geographic_Location）**：从 IP 地址解析出的物理位置
- **设备类型（Device_Type）**：用于登录的设备类别（桌面、移动、平板等）
- **浏览器（Browser）**：用于登录的网页浏览器应用程序
- **操作系统（Operating_System）**：用于登录的设备上运行的操作系统

## 需求

### 需求 1：异步登录日志记录

**用户故事：** 作为系统管理员，我希望登录尝试被异步记录，以便登录过程不会被日志操作延迟。

#### 验收标准

1. 当提交登录请求时，登录日志系统应异步触发日志记录
2. 登录日志系统应记录成功和失败的登录尝试
3. 登录日志系统在日志记录期间不应阻塞主登录流程
4. 当日志记录失败时，登录日志系统应记录错误但不影响登录结果

### 需求 2：全面的登录信息捕获

**用户故事：** 作为安全分析师，我希望获取每次登录尝试的详细信息，以便分析安全模式和调查事件。

#### 验收标准

1. 当记录登录尝试时，登录日志系统应捕获用户名
2. 当记录登录尝试时，登录日志系统应捕获源 IP 地址
3. 当记录登录尝试时，登录日志系统应从 IP 地址解析地理位置
4. 当记录登录尝试时，登录日志系统应从用户代理识别设备类型
5. 当记录登录尝试时，登录日志系统应从用户代理识别浏览器
6. 当记录登录尝试时，登录日志系统应从用户代理识别操作系统
7. 当记录登录尝试时，登录日志系统应记录登录状态
8. 当记录登录尝试时，登录日志系统应记录登录时间戳
9. 在启用多租户模式的情况下，登录日志系统应记录租户 ID

### 需求 3：数据验证和完整性

**用户故事：** 作为系统管理员，我希望登录日志数据经过验证且完整，以便数据可靠用于分析和审计。

#### 验收标准

1. 登录日志系统应确保所有必填字段在持久化前不为空
2. 登录日志系统应验证 IP 地址遵循有效的 IP 地址格式
3. 登录日志系统应验证登录状态仅包含允许的值
4. 登录日志系统应验证设备类型仅包含允许的值
5. 登录日志系统应拒绝具有无效字段值的日志记录

### 需求 4：地理位置解析

**用户故事：** 作为安全分析师，我希望看到登录尝试的地理位置，以便识别来自意外位置的可疑访问模式。

#### 验收标准

1. 当 IP 地址可以解析时，登录日志系统应使用地理位置填充位置字段
2. 如果 IP 地址无法解析为地理位置，则登录日志系统应使用"未知地址"填充位置字段
3. 登录日志系统不应将位置字段留空

### 需求 5：设备信息识别

**用户故事：** 作为安全分析师，我希望看到登录尝试的设备信息，以便识别异常设备或模式。

#### 验收标准

1. 当可以从用户代理识别浏览器时，登录日志系统应使用浏览器名称填充浏览器字段
2. 如果无法识别浏览器，则登录日志系统应使用"其他"填充浏览器字段
3. 当可以从用户代理识别操作系统时，登录日志系统应使用操作系统名称填充操作系统字段
4. 如果无法识别操作系统，则登录日志系统应使用"其他"填充操作系统字段
5. 登录日志系统不应将浏览器或操作系统字段留空

### 需求 6：可配置的数据留存

**用户故事：** 作为系统管理员，我希望配置登录日志的保留时长，以便平衡存储成本和合规要求。

#### 验收标准

1. 登录日志系统应将登录日志记录持久化到数据库
2. 登录日志系统应支持可配置的留存期限，最短为 1 年
3. 当日志记录的留存期限到期时，登录日志系统应自动删除该记录
4. 在禁用自动清理的情况下，登录日志系统不应删除过期的日志记录
5. 登录日志系统应提供配置以启用或禁用自动清理

### 需求 7：灵活的查询能力

**用户故事：** 作为安全分析师，我希望使用各种条件查询登录日志，以便调查特定事件或分析模式。

#### 验收标准

1. 登录日志系统应支持按用户名查询
2. 登录日志系统应支持按 IP 地址查询
3. 登录日志系统应支持按登录状态查询
4. 登录日志系统应支持按登录时间查询
5. 登录日志系统应支持按租户 ID 查询
6. 登录日志系统应支持同时使用多个条件的组合查询
7. 登录日志系统应支持用户名的模糊搜索
8. 登录日志系统应支持 IP 地址的模糊搜索
9. 登录日志系统应支持登录时间的时间范围查询

### 需求 8：多租户数据隔离

**用户故事：** 作为租户管理员，我希望只看到我的租户的登录日志，以便租户数据保持隔离和安全。

#### 验收标准

1. 在启用多租户模式的情况下，登录日志系统应默认按当前用户的租户 ID 过滤查询结果
2. 登录日志系统应防止用户查询属于其他租户的登录日志
3. 在当前用户是超级管理员的情况下，登录日志系统应返回所有租户的登录日志而不进行租户过滤
4. 登录日志系统应在数据访问层强制执行租户隔离

### 需求 9：主键和唯一标识

**用户故事：** 作为开发人员，我希望每条登录日志记录都有唯一标识符，以便可以单独引用和管理记录。

#### 验收标准

1. 登录日志系统应为每条登录日志记录分配唯一的 ID 作为主键
2. 登录日志系统应确保在持久化记录之前生成 ID
3. 登录日志系统应使用 ID 进行记录检索和管理操作
