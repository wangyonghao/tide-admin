# 邮件配置功能 - 后端待实现接口

## 概述
邮件配置功能已完成前端开发，需要后端实现以下接口以支持完整的验证流程。

## 需要实现的接口

### 1. 发送邮件验证码接口

**接口路径**: `POST /api/system/config/email/send-verification-code`

**功能说明**: 
- 使用用户填写的邮件配置信息发送验证码到指定邮箱
- 生成6位数字验证码
- 验证码有效期建议为5分钟
- 需要将验证码临时存储（Redis推荐），用于后续验证

**请求参数**:
```json
{
  "host": "smtp.qq.com",
  "port": 465,
  "username": "example@qq.com",
  "password": "授权码",
  "from": "系统管理员",
  "sslEnabled": true
}
```

**响应示例**:
```json
{
  "code": 200,
  "msg": "验证码已发送",
  "data": null
}
```

**业务逻辑**:
1. 接收邮件配置参数
2. 生成6位随机数字验证码
3. 使用提供的SMTP配置尝试发送邮件
4. 邮件主题：`【系统名称】邮箱验证码`
5. 邮件内容模板：
   ```
   您好，
   
   您正在配置系统邮件服务，验证码为：
   
   【123456】
   
   验证码5分钟内有效，请勿泄露给他人。
   
   如非本人操作，请忽略此邮件。
   ```
6. 将验证码存储到Redis，key格式：`email:verify:code:{username}`，过期时间5分钟
7. 如果发送失败，返回具体错误信息（如：SMTP服务器连接失败、认证失败等）

**错误处理**:
- SMTP连接失败：返回"无法连接到SMTP服务器，请检查服务器地址和端口"
- 认证失败：返回"邮箱认证失败，请检查邮箱地址和密码"
- 其他错误：返回具体的错误信息

### 2. 验证邮件验证码接口

**接口路径**: `POST /api/system/config/email/verify-code`

**功能说明**: 
- 验证用户输入的验证码是否正确
- 验证通过后允许保存邮件配置

**请求参数**:
```json
{
  "username": "example@qq.com",
  "code": "123456"
}
```

**响应示例**:
```json
{
  "code": 200,
  "msg": "验证成功",
  "data": {
    "verified": true
  }
}
```

**业务逻辑**:
1. 从Redis获取存储的验证码：`email:verify:code:{username}`
2. 比对用户输入的验证码
3. 验证成功后删除Redis中的验证码
4. 返回验证结果

**错误处理**:
- 验证码不存在或已过期：返回"验证码已过期，请重新发送"
- 验证码错误：返回"验证码错误，请重新输入"
- 验证码错误次数过多（建议限制5次）：返回"验证码错误次数过多，请重新发送"

## 安全建议

1. **频率限制**: 
   - 同一邮箱1分钟内只能发送1次验证码
   - 同一IP地址1小时内最多发送10次验证码

2. **验证码存储**:
   - 使用Redis存储，自动过期
   - 验证码加密存储（可选）

3. **错误次数限制**:
   - 同一验证码最多验证5次
   - 超过次数后需要重新发送

4. **日志记录**:
   - 记录所有验证码发送和验证操作
   - 记录失败原因，便于排查问题
